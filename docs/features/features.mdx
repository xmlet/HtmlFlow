# Features

HtmlFlow is **unopinionated** and eliminates the need for a special templating dialect.
All control flow is executed through the host language (i.e., Java), fluently chained with HtmlFlow
blocks using the `of()` or `dynamic()` builders (or `dyn` for Kotlin).

After introducing the core concepts of HtmlFlow, we present a couple of examples demonstrating some of the most common
usages with HtmlFlow. However, it’s important to **note** that there are **no limitations on the use of Java within HtmlFlow.**
